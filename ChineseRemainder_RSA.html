1 — The Problem

Find a single integer x that simultaneously satisfies k congruences x ≡ a_i (mod n_i).

Chinese Remainder Theorem

If n_1, …, n_k are pairwise coprime, the system has a unique solution modulo N = ∏ n_i.

2 — Existence (Why Coprimality Matters)

If two moduli share a factor, the corresponding residues must agree on that factor — otherwise no x can satisfy both. Pairwise coprimality removes this conflict and forces a single solution.

Counterexample: x ≡ 0 (mod 4) and x ≡ 1 (mod 6). They share the factor 2 but disagree mod 2 (0 ≠ 1) — no solution exists.

3 — Construction

Build a weighted sum that equals a_i mod n_i and zero mod every other modulus.

  • N = n_1 · n_2 · … · n_k
  • N_i = N / n_i  (divisible by every other n_j, so N_i ≡ 0 (mod n_j))
  • M_i = N_i^(-1) mod n_i  (via Extended Euclid)
CRT formula
x = ( a_1·N_1·M_1 + a_2·N_2·M_2 + … + a_k·N_k·M_k ) mod N

Reducing mod n_i, every term except the i-th vanishes, leaving a_i.

4 — Extended Euclid for Modular Inverses

The Extended Euclidean Algorithm finds integers s, t with s·a + t·n = gcd(a, n).

Bezout → inverse

If gcd(a, n) = 1, then s·a ≡ 1 (mod n), so a^(-1) ≡ s (mod n). Otherwise no inverse exists.

Iterate by dividing: q = ⌊r_prev / r_curr⌋, then update r, s, t as x_next = x_prev − q · x_curr. Stop when r = 0; the previous row holds (gcd, s, t).

5 — RSA Recap

Pick primes p, q. Set N = p·q and φ(N) = (p−1)(q−1). Choose e coprime to φ(N); let d = e^(-1) mod φ(N).

Encrypt / decrypt

c = m^e mod N  |  m = c^d mod N. Correctness follows from Euler's theorem.

Standard decryption is one full-size modular exponentiation — the bottleneck of RSA.

6 — RSA-CRT (The Speedup)

Decrypt twice in the smaller rings mod p and mod q, then recombine via CRT.

  • dP = d mod (p−1),   dQ = d mod (q−1),   qInv = q^(-1) mod p
  • m_1 = c^(dP) mod p,   m_2 = c^(dQ) mod q
  • h = qInv · (m_1 − m_2) mod p
  • m = m_2 + h · q
Why ~4× faster

Half-size operands make each multiply (b/2)² = b²/4 as expensive. Two such half-size exponentiations cost roughly ¼ of one full c^d mod N.

7 — Complexity

MethodTime
Brute-force search for xO(k · N)
CRT constructionO(k · log² max n_i)
Standard RSA decryptO((log N)³)
RSA-CRT decrypt≈ ¼ · O((log N)³)

Pseudocode
  • 1 N = product(n_i)
    Compute the combined modulus N = n_1 · n_2 · … · n_k. The unique CRT solution lies in [0, N).
  • 2 for each i: N_i = N / n_i
    N_i is divisible by every n_j with j ≠ i, so N_i ≡ 0 (mod n_j). It will isolate the i-th congruence.
  • 3 M_i = N_i^{-1} mod n_i (Extended Euclid)
    Find M_i with N_i · M_i ≡ 1 (mod n_i) using the EEA. This is the indicator multiplier.
  • 4 sum += a_i · N_i · M_i
    Add the term that satisfies congruence i and is 0 mod every other n_j.
  • 5 x = sum mod N
    Reduce to obtain the unique solution x in [0, N).
