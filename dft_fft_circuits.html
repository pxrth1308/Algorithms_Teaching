1 — The Problem

Given n complex samples x[0..n−1], compute the Discrete Fourier Transform (DFT):

X[k] = ∑(j=0..n−1) x[j] · ωj·k,  ω = e−2πi/n
Goal

Compute all n frequency bins in O(n log n) instead of the naive O(n²), using the Cooley-Tukey FFT (1965).

2 — Naive DFT

Direct evaluation: each output X[k] is a length-n dot product. n outputs × n terms each = O(n²) complex multiplies. Equivalent to the matrix-vector product X = Fn · x where Fn[j,k] = ωj·k.

3 — Roots of Unity & DFT Matrix

The n-th roots of unity ω0, ω1, …, ωn−1 are equally spaced points on the unit circle. The DFT matrix Fn[j,k] = ωj·k entries are powers of these roots; phase is −2π·jk/n.

Symmetry properties

ωn/2 = −1  and  ωk+n/2 = −ωk — the algebraic basis for the FFT split.

4 — Cooley-Tukey Decomposition

Split the input by index parity into x_even and x_odd. Compute their DFTs E[k] and O[k], then recombine:

X[k] = E[k] + ωk·O[k]    X[k+n/2] = E[k] − ωk·O[k]    for k = 0..n/2−1

Two outputs share the same E[k] and ωk·O[k] — that reuse is what collapses the cost from O(n²) to O(n log n).

5 — Bit-Reversal

The iterative FFT processes the array in-place but needs the same dependency order as the recursive form. Reordering inputs by bit-reversal of the index produces exactly that order.

Why it works

Recursive splits separate even/odd indices repeatedly. Bit-reversal of i places it next to indices that share its full recursion path, so the iterative loop combines the right pairs at every stage.

6 — Butterfly Operation

The fundamental compute unit. Inputs u, v and twiddle w:

t = w · v top = u + t bottom = u − t

One complex multiply, one add, one subtract. Each stage applies n/2 butterflies; there are log₂ n stages.

7 — Complexity

MethodTimeCircuit depth
Naive DFTO(n²)O(1)
FFT (Cooley-Tukey)O(n log n)O(log n)

FFT gate count is O(n log n); depth O(log n). Each of the log₂ n stages can run all n/2 butterflies in parallel.

8 — Worked Example (N = 4)

x = [1, 1, 1, 1] (constant signal, ω = e^(-iπ/2) = -i) X[0] = 1 + 1 + 1 + 1 = 4 X[1] = 1 + ω + ω² + ω³ = 0 X[2] = 1 - 1 + 1 - 1 = 0 X[3] = 1 + ω³ + ω⁶ + ω⁹ = 0 → X = [4, 0, 0, 0] (all energy in DC bin)

FFT Pseudocode
  1. 1x ← bitReverse(x)
    Reorder indices so iterative butterflies match the recursive FFT structure.
  2. 2for s = 1 to log₂(n):
    Each stage doubles the butterfly width and merges larger frequency groups.
  3. 3  m ← 2^s
    m is the current block size: 2, 4, 8, …, N.
  4. 4  for k = 0 to n−1 step m:
    Process each block independently at this stage.
  5. 5    for j = 0 to m/2 − 1:
    Each j picks one butterfly pair inside the block.
  6. 6      t ← w · x[k+j+m/2]
    Apply twiddle factor to the lower-half element.
  7. 7      u ← x[k+j]
    Read the upper-half element before overwriting.
  8. 8      x[k+j] ← u + t
    Top output: constructive combination.
  9. 9      x[k+j+m/2] ← u − t
    Bottom output: destructive combination.
